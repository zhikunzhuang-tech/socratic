# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Quick commands

```bash
pip install -e .              # Install in editable mode
socratic --help               # Show all commands
socratic                      # Interactive subject selection → adaptive quiz
socratic -s math              # Direct entry to a subject
socratic -s math --topic 勾股定理  # Filter by topic
socratic --solve              # Free-form question, AI Socratic tutor
socratic -g --num 5           # Batch AI generate 5 questions
socratic -s math --stats      # Learning stats
socratic -s math --report     # Weekly report with trends
socratic --review             # Wrong answer review
socratic --flash              # Flashcard mode (reveal→self-grade)
socratic --book 一元一次方程   # AI generates interactive textbook chapter
socratic --path math          # Learning path: AI plans sequential curriculum
socratic --init-kb            # Auto-generate knowledge cards for all topics
socratic --persona challenging  # Switch teaching persona
socratic --kb-create mybook   # Create a knowledge base
socratic --kb-add mybook textbook.pdf  # Add document to KB (PDF/MD/TXT)
```

No test suite exists yet. No lint/type-check tooling is configured.

## Architecture

A Python CLI app for Socratic-style interactive learning. Covers 9 subjects: math, english, physics, chinese (AI-generated), biology, geography (flashcard), claude, hermes (Claude Code / Hermes Agent trivia), and cmd (Linux/vi/git commands).

### Entry point

`socratic.cli:main` — registered as the `socratic` console script in `pyproject.toml`. Parses all argparse flags, then dispatches to the appropriate mode.

### Subject split: AI-first vs cache-first

**AI-first subjects** (`math`, `english`, `physics`, `chinese`): Real-time question generation via `sgpt` CLI. No cached bank — each question is fresh. Defined by `AI_FIRST` tuple in `quiz.py:12`.

**Cache-first subjects** (`biology`, `geography`, `claude`, `hermes`, `cmd`): Pre-written seed questions (5 per subject in `cache.py:SEEDS`) plus AI-generated additions, persisted to `data/cache/<subject>.json`. When the cache runs dry, AI generates more and appends.

### Question structure

Every question is a dict with: `id`, `topic`, `grade`, `difficulty` (1-3), `question`, `answer`, `alternatives`, `steps`, `socratic_hints` (3 progressive hints), `common_errors` (dict of wrong-answer → correction), `concept_note`, `tags`.

### Core quiz loop (`quiz.py`)

Socratic-style: present question → user answers → if wrong, show progressive hint from `socratic_hints` or match `common_errors` → if still wrong, allow skip/show-answer. After correct answer, AI generates a follow-up "why" question via `sgpt`. Persists mastery updates and streak tracking after each question.

### Adaptive system (`adaptive.py`)

Per-topic mastery scores (0.0–1.0), starting at 0.5. Correct answer increases mastery (more for first-try), wrong answer decreases it. `pick_adaptive_problem()` weights weak topics higher and adjusts difficulty: mastery <0.35 gets difficulty-1 only, mastery >0.75 gets difficulty-2+ only.

### Flashcard mode (`flash.py`)

For bio/geo/cmd — question displayed → user thinks → presses Enter → answer revealed → self-grade (y/n). Shows chapter selection with progress bars. Supports AI explanation via `e` key.

### AI dependency: `sgpt`

All AI operations call the `sgpt` CLI tool via `subprocess.run(["sgpt", prompt], ...)`. This covers: question generation (two-stage: Idea Agent → Generator), solve mode, book mode, knowledge card generation, follow-up questions, and explanations. Timeout is typically 20-30 seconds. Claude/Hermes subjects have specialized prompt templates in English.

### Persistence (`progress.py`)

Per-subject JSON files at `~/socratic/data/progress_<subject>.json`. Tracks: streak, dates, per-topic mastery, problem history, wrong answer records. No database — everything is flat JSON.

### Knowledge system

- **`knowledge.py`**: Per-topic markdown cards at `data/knowledge/<subject>/<topic>.md`, auto-generated by `sgpt`. Displayed on first wrong answer in a quiz.
- **`kb.py`**: User-uploaded document collections at `data/kb/<name>/`. Used as context for AI question generation (`set_kb_context()` → injected into generation prompts).

### Personas (`persona.py`)

Four teaching styles: `default` (balanced), `gentle` (encouraging), `challenging` (high standards), `concise` (terse). Each defines `hint_prefix`, `praise` dict, `nudge` text, and `system_extra` for AI prompt injection.

### Other modes

- **`solve.py`**: User types a free-form question; AI acts as Socratic tutor in a conversation loop.
- **`book.py`**: AI generates a textbook chapter (concepts + embedded quiz) on a given topic.
- **`path.py`**: AI plans a sequential learning curriculum from KB chapters + mastery data, with checkpoint/resume support via `progress._path`.
- **`review.py`**: Wrong answer review ordered by error frequency.
- **`report.py`**: Weekly report with daily trend bars, mastery stats, conquest rates.

### Generate pipeline (`cache.py:_generate`)

Two-stage for quality: (1) **Idea Agent** analyzes existing questions, picks a non-overlapping focus point; (2) **Generator** produces the full question JSON. Falls back to single-stage if either stage fails. Claude and Hermes subjects use dedicated `_generate_claude()` / `_generate_hermes()` with English prompts and per-module topic maps.

### Layout

```
socratic/              # Python package
├── cli.py             # Entry point, argparse, dispatch
├── quiz.py            # Core Socratic quiz loop + follow-up questions
├── cache.py           # Question cache, seed data, AI generation pipeline
├── adaptive.py        # Per-topic mastery tracking, adaptive question selection
├── flash.py           # Flashcard mode (bio/geo/cmd)
├── progress.py        # Progress persistence, stats display
├── knowledge.py       # Auto-generated knowledge cards per topic
├── kb.py              # User-managed knowledge bases (document upload)
├── problems.py        # Subject registry + dynamic problem loader
├── persona.py         # Teaching personas
├── book.py            # AI-generated interactive textbook chapters
├── path.py            # Learning path: AI curriculum planning + checkpointing
├── solve.py           # Free-form Socratic tutoring mode
├── review.py          # Wrong answer review mode
├── report.py          # Learning reports (weekly trends, mastery, conquest)
└── utils.py           # Color ANSI codes, LaTeX→plain text, answer matching
```

## Domain vocabulary

See `CONTEXT.md` for the canonical glossary. Key terms:
- **题目 (Problem)**: A question with answer, topic, difficulty, hints, etc.
- **主题 (Topic)**: Knowledge module within a subject (e.g., "一元一次方程").
- **缓存题 (Cached question)**: Pre-written or AI-generated question persisted to JSON.
- **AI 出题 (AI-generated question)**: Real-time generated, not cached long-term.
- **闪卡模式 (Flashcard mode)**: See question → guess → reveal → self-grade.
- **正向题/反向题 (Forward/Reverse)**: Command→usage vs. scenario→command.
